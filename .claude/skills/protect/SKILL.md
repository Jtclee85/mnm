# protect — 보호 기능 회귀 점검

## 트리거
- "○○ 기능이 사라졌다/깨졌다/안 된다" 리포트.
- `parseResponse.js`, `systemPrompt.js`, `i18n.js`, `chatApiShared.js`, `index.js` 레이아웃 등 공용 파일을 수정한 뒤.

## 입력
- (있으면) 깨진 기능에 대한 사용자 설명. 없으면 전체 점검.

## 단계

1. 리포트가 있으면 CLAUDE.md **보호 기능 목록**에서 해당 기능을 특정하고, 관련 spec을 먼저 실행해 재현한다.
2. `git log --oneline -15`로 최근 변경 중 해당 기능과 접점이 있는 커밋을 찾는다. 원인 커밋을 추정만 하지 말고 코드로 확인한다.
3. 원인을 **최소 수정**으로 고친다. 주변 리팩토링 금지.
4. 전체 점검이면 `npm run test:e2e`를 실행하고, 실패한 spec을 보호 기능 목록에 매핑해 보고한다.
5. 수정 후 해당 spec + `tests/smoke.spec.js` 재실행으로 회귀 해소를 확인한다.

## 알려진 함정 (실제 발생했던 회귀)

- **인라인 낱말 클릭**: 낱말 풀이 대상 용어가 쉬운 재작성 문장 안에 원형 그대로 있어야 동작한다 (문자열 매칭 의존). 프롬프트나 파싱을 바꾸면 이 기능부터 확인.
- **튜토리얼 모달**: playwright.config.js가 `mnmHistoryResearchTutorialSeen=true`를 주입한다. 테스트가 "이유 없이" 클릭 실패하면 새 모달/오버레이가 클릭을 가로채는지 확인.
- **SSE 차단 환경**: 채팅은 스트리밍 실패 시 `/api/chat-once` fallback이 있어야 한다. chat 경로 수정 시 fallback도 함께 확인.
- **i18n 문자열**: UI 문자열을 추가하면 12개 언어 + ko-easy 전부에 추가해야 한다. 한 언어에만 추가하면 다른 언어에서 undefined가 노출된다.

## 출력
- 원인(파일:줄), 수정 내용, 재실행한 테스트 결과. 원인을 못 찾으면 확인한 가설과 배제 근거를 보고한다.
